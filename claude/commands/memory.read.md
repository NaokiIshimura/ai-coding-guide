## コンテキスト

- 現在のディレクトリ: !`pwd`

## タスク

memoryファイルを読み込み、過去のセッションで記録した情報を把握してください。

### 引数

コマンド実行時に引数でmemoryディレクトリを指定できます。

```
/memory.read [memoryディレクトリパス]
```

- 引数なし: `~/.claude/projects/` 配下の現在のプロジェクトに対応するmemoryディレクトリを使用
- 引数あり: 指定されたディレクトリパスを使用

### 実施手順

1. **memoryディレクトリの決定**
   - 引数が指定されている場合はそのパスを使用
   - 引数がない場合は以下の方法で特定する
     - 現在のディレクトリのパスを取得
     - `~/.claude/projects/<プロジェクトパスをハイフン区切りに変換>/memory/` を使用
     - 例: `/Users/foo/myproject` → `~/.claude/projects/-Users-foo-myproject/memory/`

2. **memoryファイルの読み込み**
   - `MEMORY.md` を読み込む（メインファイル）
   - ディレクトリ内のトピック別ファイル（`*.md`）をすべて読み込む

3. **内容の整理と提示**
   以下の形式でユーザーにわかりやすく提示する:

   ```markdown
   # Memoryの内容

   ## 読み込んだファイル
   - MEMORY.md
   - [トピックファイル名].md
   - ...

   ## [カテゴリ1]
   - 情報1
   - 情報2

   ## [カテゴリ2]
   - 情報
   ```

4. **読み込み完了の報告**
   - 読み込んだファイルの一覧
   - 内容の要約
   - 現在のタスクに関連する情報があれば強調して提示
