## コンテキスト

- 現在の日時（日本時間）: !`TZ='Asia/Tokyo' date +"%Y-%m-%d %H:%M:%S"`
- 現在のディレクトリ: !`pwd`

## タスク

重要な情報をmemoryファイルに保存してください。

### 引数

コマンド実行時に引数でmemoryディレクトリを指定できます。

```
/memory.write [memoryディレクトリパス]
```

- 引数なし: `~/.claude/projects/` 配下の現在のプロジェクトに対応するmemoryディレクトリを使用
- 引数あり: 指定されたディレクトリパスを使用

### 実施手順

1. **memoryディレクトリの決定**
   - 引数が指定されている場合はそのパスを使用
   - 引数がない場合は以下の方法で特定する
     - 現在のディレクトリのパスを取得
     - `~/.claude/projects/<プロジェクトパスをハイフン区切りに変換>/memory/` を使用
     - 例: `/Users/foo/myproject` → `~/.claude/projects/-Users-foo-myproject/memory/`

2. **保存する情報の確認**
   - ユーザーに保存したい情報を確認する（省略されている場合は現在のセッション内容から推定）
   - 以下のカテゴリを参考に分類する:
     - 安定したパターン・規約（複数の対話で確認済みのもの）
     - 重要なアーキテクチャの決定事項・ファイルパス・プロジェクト構造
     - ユーザーのワークフロー・ツール・コミュニケーションの好み
     - 繰り返し発生する問題の解決策・デバッグの知見

3. **既存のmemoryを確認**
   - `MEMORY.md` を読み込み、既存の内容を把握する
   - 関連するトピック別ファイルがあれば読み込む
   - 重複する内容がないか確認する

4. **情報の書き込み**
   - **重複しない**: 既存の内容と重複する場合は追記ではなく更新する
   - **MEMORY.md**: 概要や参照リンクを記載（200行以内を維持）
     - 200行を超えそうな場合はトピック別ファイルへ移動
   - **トピック別ファイル**: 詳細な情報を格納
     - ファイル名例: `debugging.md`, `patterns.md`, `architecture.md`

5. **書き込み完了の報告**
   - 更新したファイルのパス
   - 保存した情報の要約

### 保存してはいけない情報

- セッション固有のコンテキスト（現在のタスク詳細、進行中の作業、一時的な状態）
- 未検証・不完全な情報
- 機密情報（パスワード、APIキー、トークン等）
- CLAUDE.mdの指示と重複・矛盾する内容

### MEMORY.md の形式

```markdown
# Memory

## [カテゴリ1]
- 重要な情報1
- 重要な情報2

## [カテゴリ2]
- 重要な情報

詳細は [topic.md](topic.md) を参照。
```
