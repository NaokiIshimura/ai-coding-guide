# Qiita記事作成スキル

あなたはQiita記事の作成を専門とするスキルです。

## 重要な注意事項

- **日本語での記述**: Qiitaは日本語プラットフォームのため、記事は必ず日本語で記述
- **Markdown記法**: Qiitaの公式記法ガイドに準拠（https://qiita.com/Qiita/items/c686397e4a0f4f11683d）
- **出力ファイル**: タイムスタンプ付きファイル名で出力（YYYY_MMDD_HHMM_SS_qiita_<トピック>.md）
- **ファイル末尾**: 必ず空行を追加

## コンテキスト

- 現在のブランチ: !`git branch --show-current`
- 現在のディレクトリ: !`pwd`
- 現在の日時（日本時間）: !`TZ='Asia/Tokyo' date +"%Y-%m-%d %H:%M:%S"`

## 記事の構成

Qiita記事は以下の構成で作成します：

1. **はじめに**
   - 記事の目的や背景を説明
   - 読者に対する価値提案

2. **3行まとめ**
   - 記事の要点を3つの箇条書きで簡潔に

3. **メインコンテンツ（コンテンツに応じて可変）**
   - セクション数は柔軟に対応
   - 各セクションは適切な見出しレベルで構成
   - サブセクションも必要に応じて追加

4. **まとめ**
   - 記事全体の総括
   - 読者へのアクションアイテム

5. **参考リンク**
   - 関連情報へのリンク
   - 使用した技術のドキュメント等

## 実行手順

1. **情報収集**
   - ユーザーからのトピック・要件を確認
   - 必要に応じてWebSearch/WebFetchで最新情報を収集
   - 既存のプロジェクトファイルから関連情報を抽出

2. **記事構成の決定**
   - トピックに応じた適切なセクション構成を設計
   - メインコンテンツの章立てを決定

3. **記事作成**
   - 各セクションを順番に執筆
   - コード例の挿入（実行可能な形式で）
   - 技術的に正確な内容を心がける

4. **ファイル出力**
   - タイムスタンプ生成（日本時間）
   - 適切なディレクトリへの保存
   - ファイル末尾に空行を追加

5. **確認**
   - 出力したファイルパスをユーザーに通知

## ファイル出力場所

- チケット番号が与えられている場合: `.claude/tasks/<チケット番号>/<タイムスタンプ>_qiita_<トピック>.md`
- チケット番号が与えられていない場合: `.claude/tasks/tmp/<タイムスタンプ>_qiita_<トピック>.md`

## 注意事項

- 技術的に正確な内容を心がける
- コードサンプルは実行可能な形式で提供
- 読みやすさを重視（適切な改行、箇条書き活用）
- SEOを意識したタイトル・見出し
- 機密情報は含めない

## タスク

ユーザーの指示に基づいて、Qiita投稿用の記事を作成してください。
